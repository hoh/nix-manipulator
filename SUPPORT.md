# Support

Thanks for using Nix-manipulator.

## Getting help

- Check the documentation in `docs/` and the README for usage guidance.
- For questions, bug reports, or feature requests, open an issue on the
  canonical Codeberg repository:
  https://codeberg.org/hoh/nix-manipulator/issues
- GitHub issues are also accepted on the mirror:
  https://github.com/hoh/nix-manipulator/issues

## What to include in an issue

- Nix-manipulator version (`nima --version` or the package version)
- Python version
- Nix version and operating system
- A minimal Nix snippet that reproduces the issue
- The exact command you ran and the expected vs actual output
- Whether the input follows RFC-166 formatting (if relevant)

## Security issues

Please report security vulnerabilities privately, as described in
`SECURITY.md` (email: git@hugoherter.com). Do not open public issues
for security problems.

## Response expectations

Support is best-effort and community-driven. Responses may take time,
so detailed reports help speed up triage.

## Professional support

If you need dedicated or time-sensitive help, professional support can be
discussed with the author.
